# fordb MA6 — Objects (Views / Routines / Triggers) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Browse views/functions/triggers by category in the schema tree, show each object's definition, and create/replace/drop views.

**Architecture:** New engine-gated `ObjectBrowser` capability (kinds + list + definition). View CRUD reuses `SchemaEditor`/`applyDdl` via new `createView`/`dropView` DdlChange kinds. The schema tree gains category folders; a new `'object'` tab shows a definition read-only.

**Tech Stack:** TypeScript strict, `pg`, `@libsql/client`, React 19 + react-arborist, Zustand, vitest, Playwright.

## Global Constraints

- pnpm; Node >= 22; TypeScript strict; no `any` except typed RPC/DB-boundary casts.
- Object introspection is read-only; schema/name bound as parameters (no interpolation). Generated view DDL quotes identifiers; the SELECT body is user SQL, preview-gated.
- Capability-gated (`adapter.objects`, `SchemaOps.createView/dropView`): the UI only offers supported kinds/ops.
- `@shared/*` alias. Renderer-importing tests → `tsconfig.web`; pure/db-host → `tsconfig.node`.
- Each task ends `pnpm typecheck && pnpm lint && pnpm test` (+ `pnpm build` for renderer; contract tasks `pnpm db:up && pnpm test:contract && pnpm db:down`). One PR per task against `main`.

- [ ] **Step 2: SQLite object browser**

`sqlite-sql.ts` add:

```ts
export const listObjects = (schema: string, type: 'view' | 'trigger'): string =>
  `SELECT name FROM ${q(schema)}.sqlite_master WHERE type = '${type}' AND name NOT LIKE 'sqlite_%' ORDER BY name`
export const objectDefinition = (schema: string, type: 'view' | 'trigger', name: string): string =>
  `SELECT sql FROM ${q(schema)}.sqlite_master WHERE type = '${type}' AND name = ${/* bind */ '?'} LIMIT 1`
```

(`type` is a fixed literal from a closed set — safe; `name` is bound via `execute({sql, args})`.) `sqlite-objects.ts`:

```ts
import type { Client } from '@libsql/client'
import type { ObjectBrowser, ObjectKind, ObjectSummary } from '@shared/adapter/object-types'
import * as SQL from './sqlite-sql'

export class SqliteObjectBrowser implements ObjectBrowser {
  readonly kinds = ['view', 'trigger'] as const
  constructor(private readonly conn: () => Client) {}
  async list(schema: string, kind: ObjectKind): Promise<ObjectSummary[]> {
    if (kind === 'function') return []
    const rs = await this.conn().execute(SQL.listObjects(schema, kind))
    return (rs.rows as unknown as { name: string }[]).map((r) => ({ name: String(r.name) }))
  }
  async definition(schema: string, kind: ObjectKind, name: string): Promise<string> {
    if (kind === 'function') return ''
    const rs = await this.conn().execute({
      sql: SQL.objectDefinition(schema, kind, name),
      args: [name]
    })
    return String((rs.rows[0] as unknown as { sql?: string } | undefined)?.sql ?? '')
  }
}
```

Wire `readonly objects = new SqliteObjectBrowser(() => this.conn)` on `SqliteAdapter`.

it('object browser: kinds, list views/triggers, definition, view create/drop', async () => {
  if (!adapter.objects || !adapter.schemaEditor) return

    buildDdl({ kind: 'createView', schema: s, name: 'ma6_v', select: `SELECT 1 AS one` }, dialect)

    buildDdl({ kind: 'dropView', schema: s, name: 'ma6_v' }, dialect)
  )
  expect((await adapter.objects.list(s, 'view')).some((v) => v.name === 'ma6_v')).toBe(false)

  private objs(id: ConnectionId): ObjectBrowser {
    const o = this.registry.get(id).objects
    if (!o) throw new Error('Object browsing is not supported by this engine')
    return o
  }
  async objectsSupported(id: ConnectionId): Promise<boolean> {
    return this.registry.get(id).objects != null
  }
  async objectKinds(id: ConnectionId): Promise<ObjectKind[]> {
    return [...this.objs(id).kinds]
  }
  listObjects(id: ConnectionId, schema: string, kind: ObjectKind): Promise<ObjectSummary[]> {
    return this.objs(id).list(schema, kind)
  }
  objectDefinition(id: ConnectionId, schema: string, kind: ObjectKind, name: string): Promise<string> {
    return this.objs(id).definition(schema, kind, name)
  }

### Task 5: Store + tree model — object nodes, definition tab, view CRUD

- [ ] **Step 1: Tree model — category + object nodes**

Extend `NodeKind` to `'schema' | 'category' | 'table' | 'view' | 'function' | 'trigger' | 'column'`. `TreeNode` gains optional `objectKind?: ObjectKind` and `category?: ObjectKind | 'table'`. `buildTree` recursion: a `category` node is expandable (its children are object/table nodes); object nodes (`view`/`function`/`trigger`) are leaves (click opens a definition tab, no children); table nodes keep column children. Node id scheme:

- schema `s:<schema>`
- category `cat:<schema>.<kind>` (kind ∈ table|view|function|trigger)
- table `t:<schema>.<name>` (child of `cat:<schema>.table`)
- object `obj:<schema>.<kind>.<name>`
- column `c:<schema>.<table>.<col>`

`buildTree` maps schemas → they load `cat:*` children; each category loads its objects. `invalidatedNodeId` maps `['conn',id,'tables',schema]` → `cat:<schema>.table` and adds `['conn',id,'objects',schema,kind]` → `cat:<schema>.<kind>`; columns unchanged. Add a unit test (`schema-tree-model.test.ts`) covering the category→object nesting + invalidatedNodeId mapping.

- [ ] **Step 2: introspection hooks for objects**

`keys.ts` add `objects: (connId, schema, kind) => ['conn', connId, 'objects', schema, kind]`. `introspection.ts` add `fetchObjects(qc, connId, schema, kind)` (calls `listObjects`) and `fetchObjectKinds`/a hook if needed. Extend `invalidateIntrospection` scope already covers `['conn', connId]`.

**Acceptance (follow the existing tree/menu patterns):**

- On connect, fetch `objectKinds(connId)` once. A schema expands to **category folders**: always **Tables**, plus **Views**/**Functions**/**Triggers** for the kinds the engine reports. Category expand lazy-loads: `fetchTables` for Tables (existing), `fetchObjects(...,kind)` for the others.
